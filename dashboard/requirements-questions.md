# Requirements Questions Library

## Discovery Questions by Category

### Purpose & Goals
- What's the primary decision this dashboard must support?
- What problem are you trying to solve?
- What would success look like for this dashboard?
- What actions should users take based on this data?
- Is this for monitoring, analysis, or executive reporting?

### Users
- Who is the primary audience?
- What's their technical expertise level?
- How often will they view this?
- Where will they view it (office, mobile, control room)?
- Do different roles need different views?

### Metrics
- What are your top 5 most important metrics?
- For each metric:
  - What is it measuring?
  - Where does the data come from?
  - How often does it update?
  - What's a "good" vs "bad" value?
  - What should it be compared against?
- Are there industry benchmarks?
- Do you have historical data for comparison?

### Visual Preferences (Don't Skip!)
**Theme:**
- Light mode, dark mode, or both?
- Where will people typically view this? (affects theme choice)

**Aesthetic Direction:**
- Should it feel minimal/spacious or dense/information-packed?
- Modern and vibrant, or traditional and conservative?
- Any dashboards or products you like the look of?
- Anything you specifically DON'T want?

**Brand:**
- Do you have brand colors I should use? (hex codes?)
- Required fonts?
- Logos or icons to include?
- Existing design system to match?

**Density:**
- See everything at once, or progressive disclosure?
- Quick glances or deep study?

### Data Sources
- What systems provide this data?
- Is data real-time or batch processed?
- Any data quality concerns?
- Expected data volume?
- Any privacy/security constraints?

### Design Constraints
- Browser/device requirements?
- Accessibility requirements?
- Framework constraints?

### Features
- Do users need to filter data?
- Should they drill down for details?
- Export functionality needed?
- Alerts/notifications required?
- User customization needed?

## Completeness Checklist

### Functional (Required)
- [ ] Purpose/decision identified
- [ ] Users and context known
- [ ] 3-5 key metrics defined
- [ ] Comparison context for each metric
- [ ] Data refresh requirements

### Visual (Required — Don't Skip!)
- [ ] Theme preference (light/dark)
- [ ] Aesthetic direction
- [ ] Brand guidelines gathered (or confirmed flexible)
- [ ] Density preference
- [ ] Inspirations or anti-inspirations noted

### Constraints (If Applicable)
- [ ] Technical requirements
- [ ] Accessibility level
- [ ] Consistency requirements
